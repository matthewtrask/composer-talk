What is Composer?

Composer is the PHP package manager allowing developers to lock dependencies together.

I say "the" because PEAR has been deemed dead.

It allows developers across the world to pull in the same packages and libraries at the same version.

All with a simple command from the command line.

mtrask@Matts-MacBook-Pro-2 ‹ refactor3 ↑ › composer install

Don't be alarmed, this is just an Oh-My-Zsh theme I use. Its called 'mortalscumbag' if you are curious.

Through Packagist, developers can browse thousands of packages to pull into an app to build it up.

Composer.json & Composer.lock

The .json file is how you tell composer and other developers exactly what you want.

The .lock file hard locks your dependencies once you run composer install.

If you run 

mtrask@Matts-MacBook-Pro-2 : ~ [0] % composer install
it will:
  • Check if a composer.lock exists
  • If not, perform a composer update to create one
  • If composer.lock exists, install the specified versions from the lock file

If you run 

mtrask@Matts-MacBook-Pro-2 : ~ [0] % composer update
it will:
  • Check composer.json
  • Determine the latest versions to install based on your version specs
  • Install the latest versions
  • Update composer.lock to reflect the latest versions installed

Having a composer.lock file also ensures consistency across clusters of servers, if you’re running Composer separately on each machine. It enables you to spin up new instances weeks, or months later, without being concerned about dependency mismatch. - Davey Shafik
